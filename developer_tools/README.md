# Developer Tools

This directory contains development utilities split into two active toolsets:

- `motion_designer/` for building motion sequences and exporting `.pvt`
- `task_gui/` for task scheduling and real-time PVT streaming via ZMQ

## Current Folder Layout

- `motion_designer/`
  - `motion_designer.py`
  - `pvt_generator.py`
  - `motions.json`, `motions.pvt`, `motions_pvt.csv`
  - `motions_csv/` (sample CSV inputs)
  - `resampled/` (intermediate outputs)
  - `proto/` (prototype scripts)
- `task_gui/`
  - `task_gui.py`
  - `config.json`
  - `csv_viewer.py`
  - `pvt_stream_viewer.py`
  - `scara_arm_csv_files/` (Pickup/Place/Home task CSV package)
- `__pycache__/`

## 1) `motion_designer/motion_designer.py`

A GUI editor to build a motion list (`Index`, `Type`, `Details`) and export motion commands through `pvt_generator.py`.

### Configuration / Parameters

This tool does not use `task_gui/config.json`.  
All motion parameters are set directly in the GUI.

Supported motion types:

- `pvt_coor`
  - `z, x, y, c, t_arm, t_servo`
  - optional lock: `t_arm = t_servo`
- `pvt_angle`
  - `angle_1, angle_2, angle_3, angle_4, t_arm, t_servo`
  - optional lock: `t_arm = t_servo`
- `pvt_csv`
  - `csv, v_max, v_safe, acc_dec, offset_z`
  - optional auto mode: `v_safe = v_max / 10`
- `pause`
  - `t_wait`
- `repeat`
  - `from_index, to_index, how_many_times`

Behavior notes:

- Inserting `pvt_csv` can auto-insert a pre-motion `pvt_coor` based on the first row of the selected CSV.
- Export command from GUI:
  - `python3 pvt_generator.py --json <selected_json> --preview <0|1>`

### Outputs

- Saved motion list JSON from the `Save` button.
- Final `.pvt` file generated by `pvt_generator.py`:
  - `<basename>.pvt`
- Temporary expanded JSON generated during pipeline:
  - `resampled_<basename>.json` (removed after conversion in current implementation).
- Optional animation preview window when `--preview 1`.

## 2) `task_gui/task_gui.py`

Main task scheduler GUI for Pickup/Place/Shuttle/Park/Home execution.  
It scans task CSV files, converts path points into joint PVT points, and streams them via ZMQ.

### Configuration (`task_gui/config.json`)

Config keys:

- `motions_dir`
  - Base folder to scan task CSV files.
- `z_by_task`
  - `pickup.z_base_mm`, `pickup.z_step_mm`
  - `place.z_base_mm`, `place.z_step_mm`
  - `shuttle.z_base_mm`, `shuttle.z_step_mm`
  - `park.z_base_mm`, `park.z_step_mm`
- `pvt`
  - `dt_ms`
- `profile`
  - `step_mm`
  - `angle_threshold`
  - `acc_dec_ms`
  - `v_max`
  - `v_safe`
  - `offset_z_global`
- `straight_pre_motion`
  - `enable`
  - `yaw_arc_radius_mm`
  - `min_distance_mm`
- `zmq`
  - `endpoint`
  - `mode` (`PUB_BIND`, `PUB_CONNECT`, `PUSH_CONNECT`)
  - `dry_run`

Runtime config source:

- Environment variable `TASK_GUI_CONFIG` (if set)
- otherwise `task_gui/config.json`

### Outputs

- Real-time ZMQ messages with payload:
  - `{"command":"pvt_point","p1","v1","p2","v2","p3","v3","p4","v4","t_ms"}`
- Runtime logs shown in the GUI (`SCAN`, `PLAN`, `DONE`, `RUN` states).
- Saved config JSON when using `Settings -> Save`.
- No automatic trajectory file export by default.

## 3) `task_gui/pvt_stream_viewer.py`

ZMQ stream monitor for incoming `pvt_point` messages.

### Inputs / Arguments

- Connects to ZMQ endpoint (`SUB` or `PULL`, depending on mode).
- Parses JSON message fields:
  - `command`, `p1`, `v1`, `p2`, `v2`, `p3`, `v3`, `p4`, `v4`, `t_ms`

### Outputs

- Live 3D visualization of reconstructed XYZ trajectory.
- Stream rate and status monitoring.
- Optional CSV logging of received stream data.

---

## 4) `task_gui/csv_viewer.py`

Standalone PyQt6 + Matplotlib utility to inspect CSV trajectory files (`X`, `Y`, `Z`) in 3D.

### Inputs

- CSV file selected via GUI.
- Required columns: `X`, `Y`, `Z`.
- Optional downsample factor from UI.

### Outputs

- Interactive 3D plot with start/end markers.
- On-screen file/row info.
